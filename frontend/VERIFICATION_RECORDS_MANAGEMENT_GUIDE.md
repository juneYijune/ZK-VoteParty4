# 验证记录管理功能使用指南

## 功能概述

验证记录管理功能允许用户查看和管理他们的投票资格验证记录。这些记录存储在浏览器的 localStorage 中，用于避免重复验证。

## 访问方式

### 方法 1: 通过侧边栏菜单

1. 登录系统后，在左侧导航栏找到"设置"菜单项（带齿轮图标）
2. 点击"设置"进入用户设置页面
3. 在设置页面中可以看到"验证记录管理"卡片

### 方法 2: 直接访问 URL

访问 `/user/settings` 页面

## 功能说明

### 1. 查看验证记录

验证记录表格显示以下信息：

| 列名 | 说明 |
|------|------|
| 投票ID | 投票活动的唯一标识 |
| 验证状态 | 显示"已验证通过"（绿色）或"不符合资格"（红色） |
| 使用的VC ID | 验证时使用的可验证凭证ID |
| 验证时间 | 完成验证的时间戳 |
| 操作 | 清除该记录的按钮 |

### 2. 清除单个验证记录

**步骤**:
1. 在验证记录表格中找到要清除的记录
2. 点击该记录行的"清除"按钮
3. 在弹出的确认对话框中点击"确认"
4. 记录被清除，下次查看该投票时需要重新验证

**确认提示**:
```
确认清除此验证记录？
清除后，下次查看该投票时需要重新验证。
```

### 3. 清除所有验证记录

**步骤**:
1. 点击页面右上角的"清除所有记录"按钮（红色按钮）
2. 在弹出的确认对话框中点击"确认清除"
3. 所有验证记录被清除

**确认对话框**:
```
确认清除所有验证记录？
此操作将清除您所有的投票资格验证记录，下次查看投票时需要重新验证。

[确认清除] [取消]
```

### 4. 退出登录时的清除选项

当您点击退出登录时，系统会自动检测是否有保存的验证记录：

#### 情况 A: 有验证记录

系统会显示确认对话框：

```
退出登录
检测到您有保存的验证记录，是否同时清除这些记录？

[清除并退出] [保留记录并退出]
```

- **清除并退出**: 清除所有验证记录，然后退出登录
- **保留记录并退出**: 保留验证记录，仅退出登录（下次登录时记录仍然有效）

#### 情况 B: 没有验证记录

直接退出登录，不显示任何提示。

## 使用场景

### 场景 1: 定期清理

如果您长时间使用系统，可能会积累很多验证记录。定期清理可以：
- 释放浏览器存储空间
- 移除过期投票的验证记录
- 保持记录列表的整洁

**建议**: 每月清理一次所有验证记录

### 场景 2: 更换 VC

如果您获得了新的 VC 或者旧的 VC 被更新，您可能需要：
1. 清除特定投票的验证记录
2. 使用新的 VC 重新验证

**操作**: 清除单个验证记录

### 场景 3: 共享设备

如果您在共享设备上使用系统：
1. 退出登录时选择"清除并退出"
2. 确保您的验证记录不会被其他人看到

**安全提示**: 在公共设备上始终选择清除验证记录

### 场景 4: 隐私保护

验证记录包含您的投票参与信息，如果您希望：
- 不留下任何痕迹
- 保护隐私

**操作**: 定期清除所有验证记录

## 技术说明

### 数据存储

验证记录存储在浏览器的 localStorage 中：

```javascript
// localStorage key 格式
zk_verifications_{wallet_address}

// 数据结构
{
  "123": {  // vote_id
    vote_id: 123,
    wallet_address: "0x...",
    vc_id: 456,
    proof: { ... },
    publicSignals: [...],
    is_verified: true,
    verified_at: "2026-01-26T10:30:00.000Z"
  }
}
```

### 数据安全

- ✅ 验证记录仅存储在本地浏览器
- ✅ 不会发送到服务器
- ✅ 清除浏览器数据会自动删除所有记录
- ✅ 不同钱包地址的记录完全隔离

### 跨设备同步

⚠️ **注意**: 验证记录不会跨设备同步

- 在设备 A 上的验证记录不会出现在设备 B 上
- 每个设备需要独立验证
- 这是为了保护隐私和安全

## 常见问题

### Q1: 清除验证记录后会发生什么？

**A**: 清除验证记录后：
- 下次查看该投票时，需要重新进行资格验证
- 您的 VC 和投票资格不受影响
- 只是需要重新生成零知识证明

### Q2: 验证记录会过期吗？

**A**: 验证记录本身不会自动过期，但：
- 如果投票已结束，该验证记录就没有实际用途了
- 如果您的 VC 被撤销或更新，旧的验证记录可能无效
- 建议定期清理不需要的验证记录

### Q3: 清除验证记录会影响我的投票吗？

**A**: 不会。清除验证记录：
- ✅ 不影响您已经投过的票
- ✅ 不影响您的投票资格
- ✅ 不影响您的 VC
- ❌ 只是需要重新验证才能查看投票详情

### Q4: 我可以恢复已清除的验证记录吗？

**A**: 不可以。验证记录一旦清除就无法恢复。但您可以：
- 重新进行资格验证
- 生成新的验证记录

### Q5: 为什么退出登录时要提示清除记录？

**A**: 这是为了：
- 保护您的隐私
- 在共享设备上防止信息泄露
- 给您选择权：保留或清除

### Q6: 验证记录占用多少存储空间？

**A**: 每条验证记录大约占用 1-2 KB 的存储空间。即使有 100 条记录，也只占用约 100-200 KB，对浏览器存储的影响很小。

## 最佳实践

### ✅ 推荐做法

1. **定期清理**: 每月清理一次所有验证记录
2. **共享设备**: 退出时选择"清除并退出"
3. **更换 VC**: 清除相关投票的验证记录后重新验证
4. **隐私保护**: 不需要的记录及时清除

### ❌ 不推荐做法

1. **频繁清除**: 不要每次使用后都清除，这会导致频繁重新验证
2. **忽略提示**: 在共享设备上不要忽略退出登录时的清除提示
3. **手动修改**: 不要尝试手动修改 localStorage 中的验证记录
